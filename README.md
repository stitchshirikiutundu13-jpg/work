## この半期で自分がどうなりたいか

**メンバーを育成し、自分が休暇を取得してもチームが回る状態を作れるリーダーになる。**

### ロールモデルと理想像

#### ロールモデル

１人目は新人の頃にお世話になった先輩。
- 周囲への気配りに優れ、すぐに声がけをし、一緒に親身になって考えてくれる。メンバーの成長を支援し、自立を促す姿勢がある。

2人目は現在の協力会社様。
- 技術に対して貪欲で、新しい技術を積極的に活用し、質問に対して丁寧に受け答えしてくれる。知らないことでもすぐに調べて問題解決してくれる技術力の高さがある。

#### 理想像と今期のフォーカス

チームメンバーに寄り添い、主体的に声がけ・サポートができるリーダー。
メンバーを育成し、自分が不在でもチームが機能する状態を作れるリーダー。

今期は、特にロールモデル1人目（先輩）の**「メンバーへの寄り添い・育成力」**に焦点を当てて取り組む。
ロールモデル2人目（協力会社様）の技術力は、今後の目標として引き続き目指していく。

### 現状の課題

- 優秀な協力会社様が**6月末にチームを抜ける予定**であり、以降は自分と1月末参加の協力会社様の2名体制になる
- 1月末参加の協力会社様は**主要Webアプリの仕様理解がほとんどできておらず、初心者レベル**の状態
- 前期は2案件を対応し、遅延は出なかったものの**残業時間が大幅に増加**した（私: 月平均約35時間、ピーク月は約80時間。メンバー: 月平均25時間以下）。持続可能な体制とは言えない
- 優秀な協力会社様が在籍している**6月末までに知見の引き継ぎを進め**、それ以降もチームが機能する状態を作る必要がある

### 方針

今期は**チーム運営（育成）に注力**する。
優秀な協力会社様が在籍している6月末までを「集中引き継ぎ期間」、7月以降を「自立運用期間」と位置づけ、半期末には「私が休暇を取得してもチームが回る」状態を目指す。

#### タイムライン

| 期間 | フェーズ | 主な取り組み |
| --- | --- | --- |
| 4月〜6月 | 集中引き継ぎ期間 | 仕様説明セッション、ペアプロ・コードリーディング、ナレッジTips整備、優秀な協力会社様との1次/2次レビュー体制で指摘差を記録 |
| 7月〜9月 | 自立運用期間 | 協力会社様が自立してタスクに取り組む。蓄積したレビュー観点・チェックリストを活用し、私が1人でレビューを担う体制に移行 |

## ■ 大目標: チーム運営の強化

### 背景・課題

- 優秀な協力会社様が6月末にチームを抜ける。7月に新規の協力会社様が参加するが、仕様を理解しているのは自分と1月末参加の協力会社様の2名になる
- 1月末参加の協力会社様は初心者レベルで、主要Webアプリの仕様理解もほとんどできていない
- 前期実績: 2案件を対応し、遅延は出なかったものの残業時間が大幅に増加（私: 月平均約35時間/ピーク月約80時間、メンバー: 月平均25時間以下。持続可能な体制ではない）
- このままでは優秀な協力会社様が抜けた後にチームが機能しなくなるリスクが高い
- 最終目標: **私が休暇を取得してもチームが回る状態**を作ること

### ◇ 行動目標

**行動目標1: 協力会社様の育成（仕様理解・コードリーディングの習得支援）**

育成観点: 主要アプリの仕様を理解し、既存コードの構造を把握した上で、改修箇所を自分で特定できるようになること

- 主要機能ごとに**仕様説明セッション**を実施し、協力会社様が各機能の目的・仕様・画面遷移を理解できるようにする
- 優秀な協力会社様が在籍している**6月末までにペアプログラミング・コードリーディングの機会**を積極的に設け、知見の引き継ぎを行う
- 開発ナレッジTips（よくあるエラー対応、開発効率化ノウハウ、環境固有の注意点など）を整備し、協力会社様が**自力で問題解決できる環境**を作る
- まとめたナレッジは**チーム共有ドキュメントとしてアウトプット**する

**行動目標2: 協力会社様の育成（設計・実装・テストの段階的な自立支援）**

育成観点: 中規模の改修タスク（工数3人日以上の改修を目安）を、設計→実装→テストまで一人で完遂できるようになること

- 小規模タスク → 中規模タスクへ**段階的に難易度を上げ**、各段階で自立度を確認しながら進める
- **設計レビュー → 実装レビュー → テストレビュー**の各段階でフィードバックを行い、指摘事項を共有ドキュメントに蓄積する
- 自分の1次レビューと優秀な協力会社様の2次レビュー（6月末まで）の**指摘差を記録**し、レビュー観点の漏れを分析・改善する
- 6月末以降は蓄積したレビュー観点をもとに**レビューチェックリスト**を整備し、自分一人でも漏れのないレビューを行える体制に移行する

**行動目標3: 遅延を出さないチーム運営の仕組みを構築する**

- タスクの担当者・ステータス・期限を**見える化**し、チーム内で日次確認を行う
- 遅れの兆候を早期に検知し、**予定の2割遅れた時点**で上司・お客様にアラートを上げる
- お客様への進捗報告について、遅れや懸念点も早い段階で共有することを習慣にする
- 前期実績（2案件/残業大幅増）をベースラインとし、**今期の案件数・遅延件数・残業時間を記録して改善を見える化**する

### □ 達成基準

- **達成基準1（育成・仕様理解）:** 協力会社様が改修タスク着手時に、対象機能の仕様確認・改修箇所の特定を**質問なしで正しく判断できた実績が3件以上**ある。開発ナレッジTipsを**10件以上**整備し、協力会社様がTipsを参照して**自力で問題解決した実績**がある
- **達成基準2（育成・自立）:** 協力会社様が中規模の改修タスク（工数3人日以上）を設計→実装→テストまで一人で完遂した実績が**1件以上**ある。レビュー指摘事項の共有ドキュメントが整備され、同じ指摘の繰り返しが減少している
- **達成基準3（チーム運営）:** 前期と比較して、残業時間が改善していることを**数値で示せる**（前期: 私 月平均約35時間/ピーク月約80時間 → 今期: 月平均30時間/ピーク月40時間）。遅延件数の記録があり、進捗管理の仕組みが構築・運用され、漏れのない状態が維持されている

行動を起こしたことでどのくらい改善したのかを、Before/Afterで見える化する。

| チーム運営 | 2案件対応/私:月平均約35h・ピーク月約80h | 遅延なし・残業時間の改善（月平均・ピーク月ともに減少） | 案件数、遅延件数、残業時間の月次記録 |
